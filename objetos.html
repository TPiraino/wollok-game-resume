🎨 Objetos Visuales

Aprende a agregar sprites, imágenes y elementos visuales a tu juego

🎭 ¿Qué son los Objetos Visuales?

Los objetos visuales son elementos que aparecen en el tablero del juego. Para que un objeto sea visual, debe entender el mensaje position() y retornar la posición donde se quiere mostrar.

🎯 Concepto Clave

Un objeto visual = Un objeto que sabe dónde está ubicado en el tablero

📍 El Método position()

Todo objeto visual debe implementar el método position() que retorna la posición donde se quiere mostrar:

object miObjeto {
    var posicion = game.at(2, 3)
    
    method position() = posicion
}

Usando position como propiedad

object miObjeto {
    const position = game.at(2, 3)  // Más simple
}

🖼️ Visuales (Sprites)

Los visuales definen qué imagen mostrar. Un objeto puede tener múltiples visuales para diferentes estados:

object personaje {
    const position = game.at(5, 4)
    
    method visual() = "personaje.png"
    
    // O múltiples visuales
    method visual() {
        if (estaCorriendo()) {
            "personaje-corriendo.png"
        } else {
            "personaje-quieto.png"
        }
    }
}

🎨 Tipos de Visuales

  • Personajes: Protagonistas del juego
  • Enemigos: Obstáculos y adversarios
  • Objetos: Items, monedas, power-ups
  • Decoración: Elementos del escenario

➕ Agregar Objetos al Juego

Para que un objeto aparezca en el tablero, debes agregarlo usando game.addVisual():

import wollok.game.*

program miJuego {
    // Crear objetos
    const personaje = new Personaje()
    const enemigo = new Enemigo()
    const moneda = new Moneda()
    
    // Agregar al tablero
    game.addVisual(personaje)
    game.addVisual(enemigo)
    game.addVisual(moneda)
    
    game.start()
}

🏗️ Ejemplo Completo

Un ejemplo completo con diferentes tipos de objetos:

import wollok.game.*

// Definir clases
class Personaje {
    var position = game.at(2, 3)
    method visual() = "hero.png"
}

class Enemigo {
    var position = game.at(8, 5)
    method visual() = "enemy.png"
}

class Moneda {
    var position = game.at(5, 2)
    method visual() = "coin.png"
}

program juegoCompleto {
    // Configurar tablero
    game.width(10)
    game.height(7)
    game.title("Mi Juego")
    
    // Crear instancias
    const hero = new Personaje()
    const villain = new Enemigo()
    const gold = new Moneda()
    
    // Agregar al juego
    game.addVisual(hero)
    game.addVisual(villain)
    game.addVisual(gold)
    
    game.start()
}

🎭 Objetos con Múltiples Estados

Puedes cambiar la apariencia de un objeto según su estado:

class Personaje {
    var position = game.at(3, 4)
    var corriendo = false
    var herido = false
    
    method visual() {
        if (herido) {
            "personaje-herido.png"
        } else if (corriendo) {
            "personaje-corriendo.png"
        } else {
            "personaje-normal.png"
        }
    }
    
    method correr() {
        corriendo = true
    }
    
    method parar() {
        corriendo = false
    }
}

👻 Objetos Invisibles

Puedes crear objetos que no se vean pero que estén presentes en el juego:

class TriggerInvisible {
    var position = game.at(5, 5)
    
    method visual() = ""  // Sin imagen = invisible
    
    method activar() {
        console.println("¡Trigger activado!")
    }
}

🎯 Usos de Objetos Invisibles

  • Triggers: Activar eventos al pasar por encima
  • Zonas de colisión: Áreas especiales del mapa
  • Puntos de spawn: Donde aparecen enemigos

🔄 Actualizar Visuales

Para que los cambios en los visuales se reflejen, el juego se actualiza automáticamente. Si necesitas forzar una actualización:

// El juego se actualiza automáticamente
// Pero puedes forzar actualización si es necesario
game.refresh()

💡 Consejos Importantes

  • Nombres de archivos: Usa nombres descriptivos para tus imágenes
  • Tamaño de imágenes: Deben caber en las celdas del tablero
  • Formatos soportados: PNG, JPG, GIF
  • Ruta de imágenes: Colócalas en la carpeta del proyecto
  • Performance: No agregues demasiados objetos a la vez
