📚 Introducción a Wollok Game

Todo lo que necesitas saber para empezar a crear juegos

¿Qué es Wollok Game?

Wollok-Game es una biblioteca de Wollok que sirve para crear juegos. La idea que propone es tener una especie de "tablero" en donde se puedan ubicar los objetos de algún programa de Wollok y éste se mostrará automáticamente en la pantalla.

🎯 Concepto Clave

Imagina un tablero de ajedrez donde puedes colocar objetos y estos se muestran automáticamente. ¡Eso es Wollok Game!

🚀 Configuración Inicial

Para usar Wollok Game, necesitas importar la biblioteca al inicio de cada archivo:

import wollok.game.*

El objeto más importante: game

El objeto game es tu punto de entrada principal. Con él puedes:

  • Iniciar el juego
  • Configurar el tablero
  • Agregar objetos visuales
  • Manejar eventos

🎮 Iniciar un Juego

Para comenzar un juego y visualizar el tablero, simplemente ejecuta:

game.start()

Esto abrirá una ventana independiente mostrando el tablero del juego.

📝 Formas de Usar Wollok Game

1. Por Consola (Sin archivo)

>>> import wollok.game.*
>>> game.start()

2. Con archivo .wlk

import wollok.game.*

object prueba {
    method iniciar() {
        game.start()
    }
}

Luego en la consola: prueba.iniciar()

3. Con programa .wpgm (Recomendado)

import wollok.game.*

program ejemplo {
    game.start()
}

Se ejecuta directamente sin abrir la consola.

⚙️ Configuración Básica del Tablero

Antes de iniciar el juego, puedes configurar sus propiedades básicas:

import wollok.game.*

program miJuego {
    game.width(10)        // Ancho en celdas
    game.height(7)        // Alto en celdas
    game.cellSize(50)     // Tamaño de cada celda en píxeles
    game.title("Mi Juego") // Título de la ventana
    
    game.start()
}

📏 Propiedades del Tablero

  • width(): Ancho del tablero en celdas
  • height(): Alto del tablero en celdas
  • cellSize(): Tamaño de cada celda en píxeles (default: 50x50)
  • title(): Título de la ventana del juego
  • ground(): Imagen de fondo para las celdas

🎯 Características del Tablero

  • Único: Solo puede haber un tablero por juego
  • Dividido en celdas cuadradas: Todas las posiciones están en unidades de celdas
  • No se puede estar "en el medio" de una celda
  • Configurable: Puedes cambiar tamaño, título y fondo
